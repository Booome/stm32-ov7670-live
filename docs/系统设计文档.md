## 1. OV7670 DMA 采集架构

### 1.1 硬件连接

**MCU：STM32F103C8T6（72MHz，LQFP48）**

| 信号 | STM32 引脚 | 功能 | 方向 |
|------|------------|------|------|
| OV7670_D0 ~ D7 | PA0 ~ PA7 | 8-bit 数据总线（FIFO_A0~A7） | 输入 |
| FIFO_RCK | PB1 → TIM3_CH4 | 读时钟 | 输出 |
| FIFO_OE | PA8 | 输出使能，低有效 | 输出 |
| FIFO_WR_EN | PB5 | 写使能 | 输出 |
| FIFO_WRST | PB7 | 写指针复位 | 输出 |
| FIFO_RRST | PB6 | 读指针复位 | 输出 |
| OV7670_VSYNC | PA11 → EXTI11 | 帧同步信号 | 输入 |
| OV7670_HREF | PA12 | 行有效 | 输入 |
| OV7670_SCL | PB10 | SCCB 时钟（模拟 I²C） | 输出 OD |
| OV7670_SDA | PB11 | SCCB 数据（模拟 I²C） | 双向 |
| OV7670_PWDN | PB3 | 掉电控制 | 输出 |
| OV7670_RESET | PB4 | 复位 | 输出 |

### 1.2 RCK 频率选择

**目标**：160×128 RGB565 @ 30fps，最低需 1.23 MHz。

**双 DMA 约束**：Camera DMA 填满半个 buffer 时间 > SPI DMA 发送半 buffer 时间。

```
条件：320 / RCK  >  320 / SPI_rate
即：  RCK        <  SPI_rate ≈ 2.25 MB/s  (SPI @ 18 MHz)
```

| RCK 频率 | ARR | DMA 320B | SPI 320B | 余量 | 满足 30fps |
|----------|-----|----------|----------|------|------------|
| 2.0 MHz | 35 | 160 μs | 142 μs | 18 μs | +63% |
| 1.8 MHz | 39 | 178 μs | 142 μs | 36 μs | +46% |
| **1.44 MHz** | **49** | **222 μs** | **142 μs** | **80 μs** | **+17%** |

**最终选择：RCK = 1.44 MHz**

- ARR = 49, Pulse = 24（≈50% 占空比）
- Camera 端：帧读时间 28.4ms，绝对余量 4.9ms（+17%），覆盖行消隐绰绰有余
- SPI 端：DMA 320B 耗时 222μs，SPI 发送 142μs，余量 80μs（+120%），大幅安全

## 2. 双 DMA 架构

### 2.1 核心思路

两个 DMA 独立运行、各司其职，帧内 Camera DMA 全程不停，全程零拷贝：

```
TIM3_CH4   Camera DMA (Ch3)     SPI DMA (Ch5)
   │            │                    │
   ▼            ▼                    ▼
[RCK]       OV7670_DATA_PORT->IDR ──► CameraBuffer[640] ──► SPI2-DR ──► LCD
  PWM        Circular             Normal (每次 320B)
           HT/TC 中断触发
```

- **Camera DMA**：DMA1_Ch3, Circular, Peripheral→Memory，RCK 每拍自动搬运 1 字节到 CameraBuffer[640]
- **SPI DMA**：DMA1_Ch5, Normal, Memory→Peripheral，从 CameraBuffer 中对应半区发送到 SPI2

**半满 = 半区可消费，全满 = 另一半区可消费，无需拷贝。**

### 2.2 CameraBuffer 布局

```
CameraBuffer (640 bytes = 2 × 320B)
┌──────────────────────┬──────────────────────┐
│    Buffer A (320B)   │    Buffer B (320B)   │
│   CameraBuffer[0]    │  CameraBuffer[320]   │
│         ...          │         ...          │
│   CameraBuffer[319]  │  CameraBuffer[639]   │
└──────────────────────┴──────────────────────┘
         ▲HT 中断               ▲TC 中断
    "半满，A 可消费"       "全满，B 可消费"
```

### 2.3 中断处理

```
HT (Half-Transfer) 中断：
  ├── SPI DMA 源地址 = &CameraBuffer[0]
  ├── 长度 = 320 字节
  └── 启动 SPI DMA 发送

TC (Transfer-Complete) 中断：
  ├── SPI DMA 源地址 = &CameraBuffer[320]
  ├── 长度 = 320 字节
  └── 启动 SPI DMA 发送
```

### 2.4 时序

```
RCK: ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
     ┃  1.44 MHz，帧内持续输出               ┃
     ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

CAM  ┌──A(320B)──┬──B(320B)──┬──A(320B)──┬──B(320B)──┬...
DMA  │   Camera  │   Camera  │  Camera   │  Camera   │
     └───────────┴───────────┴───────────┴───────────┘
          ▲HT          ▲TC         ▲HT         ▲TC

SPI             [SPI发A]    [SPI发B]    [SPI发A]    [SPI发B]
DMA             320B         320B        320B         320B

                    ▲ SPI DMA 必须在下一个 TC 前完成
```

### 2.5 帧采集状态机

```
IDLE
  │ VSYNC 下降沿中断（通过 EXTI11）
  ▼
FRAME_START
  │ RRST 负脉冲 → 复位 FIFO 读指针
  │ 设置 byte_total = 40960
  │ HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_4)
  │ HAL_DMA_Start(&hdma_cam, (uint32_t)OV7670_DATA_ADDR,
  │               (uint32_t)CameraBuffer, 640)
  ▼
FRAME_CAPTURING  ←── 循环
  │ HT 中断 → SPI DMA 发送 Buffer A (320B)
  │ TC 中断 → SPI DMA 发送 Buffer B (320B)
  │ 每次 HT/TC: byte_sent += 320
  │ byte_sent >= byte_total ?
  ▼
FRAME_DONE
  │ 等待 SPI DMA 发完最后半区
  │ HAL_TIM_PWM_Stop(&htim3, TIM_CHANNEL_4)
  │ HAL_DMA_Stop(&hdma_cam)
  ▼
IDLE
```

帧间：
```
VSYNC ↓ → RRST 复位 FIFO 读指针 → 多余 RCK 脉冲的数据被丢弃 → 下一帧
```

```c
DMA1_Channel3->CPAR = (uint32_t)OV7670_DATA_ADDR;
```

## 4. 关键代码框架

### 4.1 Camera DMA HT/TC 中断

```c
void DMA1_Channel3_IRQHandler(void) {
    if (DMA1->ISR & DMA_ISR_HTIF3) {
        DMA1->IFCR = DMA_IFCR_CHTIF3;
        HAL_SPI_Transmit_DMA(&hspi2, CameraBuffer, 320);
    }

    if (DMA1->ISR & DMA_ISR_TCIF3) {
        DMA1->IFCR = DMA_IFCR_CTCIF3;
        HAL_SPI_Transmit_DMA(&hspi2, CameraBuffer + 320, 320);
    }
}
```

### 4.2 VSYNC 中断（帧起止）

```c
void EXTI15_10_IRQHandler(void) {
    if (__HAL_GPIO_EXTI_GET_IT(OV7670_VSYNC_Pin) &&
        HAL_GPIO_ReadPin(OV7670_VSYNC_GPIO_Port, OV7670_VSYNC_Pin) == GPIO_PIN_RESET) {

        __HAL_GPIO_EXTI_CLEAR_IT(OV7670_VSYNC_Pin);

        if (state == IDLE) {
            state = FRAME_START;
            byte_sent = 0;

            HAL_GPIO_WritePin(OV7670_FIFO_RRST_GPIO_Port, OV7670_FIFO_RRST_Pin, GPIO_PIN_RESET);
            HAL_GPIO_WritePin(OV7670_FIFO_RRST_GPIO_Port, OV7670_FIFO_RRST_Pin, GPIO_PIN_SET);

            HAL_TIM_PWM_Start(&htim3, TIM_CHANNEL_4);
            HAL_DMA_Start_IT(&hdma_cam, (uint32_t)OV7670_DATA_ADDR,
                             (uint32_t)CameraBuffer, 640);
            state = FRAME_CAPTURING;
        }
    }
}
```

## 6. OV7670 SCCB 初始化配置

### 6.1 目标分辨率与缩放策略

**结论：160×128 可通过 OV7670 两层缩放架构实现，无需裁剪或黑边。**

依据 OV7670 Implementation Guide (V1.0) 第 6 章 "Image Scaler"（docs/OV7670_Implementation_Guide_V1.0.pdf），OV7670 缩放电路由两层组成：

1. **Down Sampling（DCW）**：整数下采样 1/2N（by 2 / 4 / 8），由 SCALING_DCWCTR(0x72) 控制
2. **Digital Zoom Out**：分数缩放 1x ~ 0.5x，由 SCALING_XSC(0x70) / SCALING_YSC(0x71) 控制

Digital Zoom Out 最大支持 CIF(352×288) 尺寸。两层组合可将 VGA 缩放到 CIF 以下的任意尺寸。

**缩放比例公式**（IG Section 6.4, p40）：

```
SCALING_XSC[6:0] = 0x20 × (DownSample_Width  / Target_Width)
SCALING_YSC[6:0] = 0x20 × (DownSample_Height / Target_Height)
```

其中 0x20 → 1.0x（不缩放），0x40 → 0.5x（缩半），范围 [0x20, 0x40] 对应 [1x, 0.5x]。

**160×128 缩放路径与计算**：

| 步骤 | 水平 | 垂直 |
|------|------|------|
| ① VGA 原始 | 640 | 480 |
| ② Down Sampling by 2 (DCW) | 320 | 240 |
| ③ Digital Zoom Out | → 160（0.5x） | → 128（0.533x） |

寄存器值计算：
- SCALING_XSC = 0x20 × 320 / 160 = 0x20 × 2 = **0x40**（边界值 0.5x，有效）
- SCALING_YSC = 0x20 × 240 / 128 = 0x20 × 1.875 = **0x3C**（60, ratio=0.533）

验证：320 × 0.5 = 160 ✓，240 × 0.533 = 128 ✓。**整个画面缩放，无裁剪，无变形。**

**参数影响**：`byte_total = 160 × 128 × 2 = 40960`，与原始设计第 2 章一致。40960 / 320 = 128 次半区 DMA，整数。

### 6.2 SCCB 协议时序

OV7670 通过 SCCB（兼容 I²C）经 SIO_C(PB10) / SIO_D(PB11) 配置寄存器。STM32 端用 GPIO 模拟 SCCB 时序。datasheet Table 4 关键参数：

| 参数 | 含义 | 要求 |
|------|------|------|
| f_SIO_C | 时钟频率 | ≤ 400 kHz |
| t_LOW | 时钟低电平 | ≥ 1.3 μs |
| t_HIGH | 时钟高电平 | ≥ 600 ns |
| t_HD:STA | START 保持 | ≥ 600 ns |
| t_SU:STA | START 建立 | ≥ 600 ns |
| t_SU:DAT | 数据建立 | ≥ 100 ns |
| t_HD:DAT | 数据保持 | ≥ 0 μs |
| t_SU:STO | STOP 建立 | ≥ 600 ns |
| t_BUF | 总线空闲 | ≥ 1.3 μs |
| t_R / t_F | 上升/下降沿 | ≤ 300 ns |

**写寄存器流程（2-phase write）**：

```
START → 0x42(ID+W) + ACK → reg_addr + ACK → data + ACK → STOP
```

**读寄存器流程（2-phase read）**：

```
START → 0x42 + ACK → reg_addr + ACK → RESTART → 0x43(ID+R) + ACK → data(NACK) → STOP
```

> 实现提示：STM72MHz 下，每个 SIO_C 沿之间插入约 50 个 NOP（≈0.7μs）即可满足时序。SIO_D 在 SIO_C 高电平期间保持稳定。

### 6.3 寄存器配置表（160×128 RGB565）

基于 datasheet Table 5。下表列出影响分辨率/格式/缩放的关键寄存器；AWB / gamma / 镜头校正等图像质量寄存器可参考社区成熟配置集，此处从略。

| 地址 | 寄存器 | 写入值 | 作用 |
|------|--------|--------|------|
| 0x12 | COM7 | 0x14 | bit4=1 QVGA（DCW by2 基础 → 320×240）；bit2=1,bit0=0 → RGB565 |
| 0x0C | COM3 | 0x0C | bit3=1 Digital Zoom Enable；bit2=1 Down Sampling Enable |
| 0x3E | COM14 | 0x18 | bit4=1 DCW/scaling PCLK 使能；bit3=1 手动缩放使能 |
| 0x40 | COM15 | 0xD0 | bit[7:6]=11 输出满量程 [00-FF]；bit[5:4]=01 RGB565 |
| 0x72 | SCALING_DCWCTR | 0x11 | bit[5:4]=01 垂直 by2；bit[1:0]=01 水平 by2；rounding |
| **0x70** | **SCALING_XSC** | **0x40** | **水平 0.5x** = 0x20 × 320 / 160（320→160） |
| **0x71** | **SCALING_YSC** | **0x3C** | **垂直 0.533x** = 0x20 × 240 / 128（240→128） |
| **0x73** | **SCALING_PCLK_DIV** | **0x01** | IG Table 6-3：水平总缩放 1/4x → divider=2（bit[2:0]=001） |
| **0xA2** | **SCALING_PCLK_DELAY** | **0x02** | 时序延迟（默认值，实现时按 IG 6.3 公式微调） |
| 0x11 | CLKRC | 0x00 | 内部时钟不分频 F(clk)=F(XCLK)/(0+1) |
| 0x8C | RGB444 | 0x00 | bit1=0 关 RGB444（确保走 RGB565） |
| 0x13 | COM8 | 0xC5 | bit2=AGC, bit1=AWB, bit0=AEC 使能；bit5=banding filter on |
| 0x3B | COM11 | 0x0A | bit4=1 50/60Hz 自动检测 |
| 0x14 | COM9 | 0x4A | AGC ceiling 默认（bit[6:4]=100 → 16x） |
| 0x1E | MVFP | 0x01 | Mirror/VFlip，按摄像头安装方向调整 |
| 0x3A | TSLB | 0x0D | 输出序列默认（配合 COM13） |
| 0x3D | COM13 | 0x88 | gamma enable，UV swap 默认 |

> IG 256×128 官方例子（Section 6.1, p38）与上述 160×128 使用相同的垂直缩放比例（YSC=0x3C，240→128），侧面印证公式正确。推荐先用 256×128（XSC=0x28）验证 scaler 链路，再改 XSC=0x40 到 160×128。

> **缩放验证要点**：写完上述寄存器后，OV7670 应输出 160×128 行列。可用彩条测试模式验证，每帧应输出 160×128×2 = 40960 字节。验证通过后再关闭彩条进入正常采集。

## 7. ST7735 LCD 驱动初始化

### 7.1 引脚映射

`main.h` 已定义 ST7735 控制脚，数据走 SPI2（datasheet 4-line serial 接口）：

| 信号 | STM32 引脚 | 功能 |
|------|-----------|------|
| LCD_SPI_CLK | PB13 (SPI2_SCK) | SPI 时钟 |
| LCD_SPI_MOSI | PB15 (SPI2_MOSI) | SPI 主出从入 |
| LCD_SPI_CS | PB12 | 片选，低有效 |
| LCD_DC | PB8 | 数据/命令选择（0=命令，1=数据） |
| LCD_RESET | PB14 | 硬件复位，低有效 |
| LCD_BL | PB9 | 背光使能 |

SPI2 配置（`main.c` 已生成）：主机、8bit、CPOL=0/CPHA=0、预分频 2（=18MHz）、MSB first。RGB565 16bit/pixel 时 datasheet 要求 COLMOD(3Ah)=05h（见 9.7.21）。

### 7.2 上电初始化命令序列

依 datasheet 9.12 / 9.13 电源流程：

1. LCD_BL 拉高开背光
2. LCD_RESET 拉低 ≥ 10μs 后拉高（硬件复位）
3. 延时 120ms（datasheet：复位后需 120ms 寄存器恢复）
4. `SLPOUT (0x11)` 退出睡眠，延时 120ms（等 booster 稳定）
5. `MADCTL (0x36)` + 1 参数：设内存访问方向
6. `COLMOD (0x3A)` + 0x05：16bit/pixel RGB565
7. `FRMCTR1 (0xB1)` + 帧率参数（可选）
8. `INVON (0x21)` 或 `INVOFF (0x20)`（按模块极性，1.8寸模块常需 INVON）
9. `NORON (0x13)`：正常显示模式
10. `DISPON (0x29)`：开显示

**MADCTL(0x36) 位定义**（datasheet 10.1.26）：

| bit | 名称 | 含义 |
|-----|------|------|
| D7 | MY | 行地址方向（0=上→下，1=下→上） |
| D6 | MX | 列地址方向（0=左→右，1=右→左） |
| D5 | MV | 行/列交换 |
| D4 | ML | LCD 刷新方向 |
| D3 | RGB | 0=RGB 排列，1=BGR 排列 |
| D2 | MH | 水平方向 |
| D1/D0 | — | 保留 |

> MADCTL 取值按屏幕物理安装方向调整。常见值 0x00（RGB、正向）或 0xC0（翻转）。若模块像素为 BGR 排列则 D3=1。

### 7.3 帧写入协议（CASET / RASET / RAMWR）

每帧写画面前，CPU 先发命令设置写窗口，再连续推像素数据（datasheet 9.10 / 10.1.19~21）：

```
1. CASET(0x2A) + 4 字节: XS_H XS_L XE_H XE_L    // 列地址范围
2. RASET(0x2B) + 4 字节: YS_H YS_L YE_H YE_L    // 行地址范围
3. RAMWR(0x2C)                                  // 写显存，随后为像素数据流
```

**160×128 全屏显示**（ST7735 128×160 屏通过 MADCTL MV=1 交换行列，横屏 160×128）：

| 命令 | 参数 | 含义 |
|------|------|------|
| CASET | 0x00, 0x00, 0x00, 0x9F | 列 0~159（MV 交换后对应 160 列） |
| RASET | 0x00, 0x00, 0x00, 0x7F | 行 0~127（MV 交换后对应 128 行） |

> 发命令时 DC=0；发参数与像素时 DC=1。RAMWR 之后 DC 保持 1，ST7735 连续接收 40960 字节像素直至下次命令。地址寄存器在写窗口内自动递增，越界回绕（datasheet 9.9）。

### 7.4 SPI 命令发送函数约定

LCD 命令由 CPU 阻塞发送（`HAL_SPI_Transmit`），不占用 DMA 通道：

```c
void lcd_write_cmd(uint8_t cmd);                       // DC=0, CS=0, TX cmd, CS=1
void lcd_write_data(uint8_t *buf, uint16_t len);       // DC=1, CS=0, TX buf, CS=1
```

像素数据流由 SPI DMA 发送（见第 11 章），发送期间 DC=1、CS=0 保持。

## 8. AL422B FIFO 读写时序控制

### 8.1 引脚映射与硬件连接

AL422B 是 3M-bit（393,216×8）FIFO 帧存储，作为 OV7670 与 STM32 之间的异步缓冲。写侧接摄像头，读侧接 STM32。

| AL422B 引脚 | 信号 | 连接 | 说明 |
|-------------|------|------|------|
| DI0~DI7 | 数据输入 | OV7670 D0~D7 | 写数据来自摄像头 |
| WCK | 写时钟 | OV7670 PCLK | 像素时钟驱动写入 |
| /WE | 写使能 | OV7670 HREF（或 PB5） | 低有效，仅有效行写入 |
| /WRST | 写复位 | OV7670 VSYNC（或 PB7） | 帧起始复位写指针 |
| DO0~DO7 | 数据输出 | STM32 PA0~PA7 | 读数据送 GPIOA->IDR |
| RCK | 读时钟 | STM32 PB1 (TIM3_CH4) | 1.44MHz PWM |
| /RE | 读使能 | **硬件接地** | 恒使能，用 /OE 控制输出 |
| /OE | 输出使能 | STM32 PA8 | 低有效 |
| /RRST | 读复位 | STM32 PB6 | 帧起始复位读指针 |
| TST | 测试 | 接地 | 正常工作下拉 |
| DEC | 去耦 | 3.3V + 0.1μF | 3.3V 模式 |
| VDD / GND | 电源/地 | 3.3V / GND | 引脚 10 / 6,23 |

> **关键硬件约定**：`/RE` 恒接地（读地址始终递增），仅用 `/OE` 控制输出三态。`WCK` 接 OV7670 PCLK，`/WE` 接 OV7670 HREF（仅有效行写入，消隐行不进 FIFO）。如此 STM32 侧无需控制写时序，只管读。
>
> main.h 定义了 `OV7670_FIFO_WR_EN`(PB5)、`OV7670_FIFO_WRST`(PB7)。若 PCB 上 /WE 已接 HREF、/WRST 已接 VSYNC，则 PB5/PB7 可悬空或作软件备份；若由 STM32 控制，则采集期间 PB5 保持低、PB7 每帧给负脉冲。**以 eda 目录原理图为准，实现时确认。**

### 8.2 写操作（OV7670 → FIFO）

datasheet 8.1：DI 数据在 WCK 上升沿写入，当 /WE 为低。写指针随 WCK 自动递增；/WE 高时禁止写入、写指针停住。/WRST 低脉冲复位写指针为 0（在 WCK 上升沿采样）。

```
VSYNC ↓ (帧起始)
  → /WRST 负脉冲 → 写指针=0
  → /WE 跟随 HREF：有效行期间低(写入)，消隐期间高(停)
  → PCLK 上升沿逐字节写入 FIFO
VSYNC ↓ (下一帧)
  → /WRST 再次复位，覆盖旧数据
```

### 8.3 读操作（FIFO → STM32）

datasheet 8.1：DO 数据与 RCK 同步，RCK 上升沿当 /OE 低时输出有效。访问时间 T_AC ≤ 15ns（RCK↑ 后数据就绪）。读指针随 RCK 递增；/OE 高时输出高阻（读指针仍递增）。/RRST 低脉冲复位读指针为 0（RCK 上升沿采样）。

```
VSYNC ↓ 中断
  → /RRST 负脉冲 → 读指针=0
  → /OE 拉低（持续使能输出）
  → 启动 TIM3_CH4 PWM(RCK 1.44MHz) + Camera DMA
  → 每个 RCK 上升沿，DO0~DO7 稳定后 DMA 从 GPIOA->IDR 搬 1 字节
  → HT/TC 中断触发 SPI DMA 发送
  → byte_sent 累计到 40960，停止 RCK 与 DMA
  → /OE 拉高（帧间空闲）
```

### 8.4 AC 时序参数（datasheet 7.4，3.3V 模式）

| 参数 | 含义 | Min | Max |
|------|------|-----|-----|
| T_RC | RCK 周期 | 20 ns | 1000 ns |
| T_RPH / T_RPL | RCK 高/低脉宽 | 7 ns | — |
| T_AC | 访问时间（RCK↑→数据有效） | — | 15 ns |
| T_OH | 输出保持 | 4 ns | — |
| T_RRS / T_RRH | /RRST 建立/保持 | 5 / 2 ns | — |
| T_OES / T_OEH | /OE 建立/保持 | 5 / 2 ns | — |
| T_OPW | /OE 脉宽 | 10 ns | — |
| T_TR | 信号转换时间 | 2 ns | 20 ns |

> **RCK=1.44MHz 余量校核**：周期 694ns ≫ T_RC(20ns)；高/低各 ≈347ns ≫ 7ns；T_AC(15ns) 内数据就绪，DMA 在 RCK 上升沿触发、下一周期搬运，余量充足。

### 8.5 关键约束（datasheet 8.3）

1. **WCK/RCK 须保持 ≥1MHz 自由运行**：较快的时钟作 DRAM 刷新。本项目帧内 RCK 1.44MHz 持续；帧间 RCK 停，但 WCK(=PCLK) 仍跑（摄像头持续输出），保证刷新。
2. **读地址须滞后写地址 128~393,247 周期**。VSYNC 同步下，读指针帧起始复位为 0、写指针从 0 增长，读始终落后写，满足。
3. ⚠ **/RE 不能在 /RRST 上升沿为低**（datasheet 8.3.2）。本项目 /RE 恒接地——复位 /RRST 时其上升沿 /RE 为低，**违反此约束**。

> **约束 3 风险缓解**（实现阶段须结合 PCB 确认 /RE 接法）：
> - 方案 A：硬件 /RE 改由 STM32 控制，复位 /RRST 前先拉高 /RE，复位后再拉低。
> - 方案 B：/RE 保持接地，复位 /RRST 期间确保无 RCK 边沿（PWM 未启动），且 /OE 拉高使输出高阻，降低风险。
> - 建议实现时优先用方案 A；若 PCB 已固化接地则方案 B，并实测验证。

## 9. RGB565 格式——零转换直送

OV7670 配 RGB565（COM15 bit[5:4]=01），ST7735 配 16bit/pixel RGB565（COLMOD=0x05）。两者像素格式一致，`CameraBuffer` 中每 2 字节即 1 个 RGB565 像素，SPI DMA 直接搬到 LCD 显存，**无需任何颜色/字节序转换**。

字节序匹配：OV7670 默认高字节在前（MSB first），ST7735 4-line serial 16bit 模式亦高字节在前，SPI 配 MSB first。三者一致，直送即可。

> 若实测颜色错乱（如红蓝互换）：检查 OV7670 COM3 bit6（输出 MSB/LSB swap）、ST7735 MADCTL D3（RGB/BGR）。调整其一即可，仍不需软件转换。故第 5 章原"RGB565→LCD 格式转换（如需要）"项标记为**不需要**。

## 10. HAL 回调统一

> 第 4 章代码框架用裸寄存器操作 `DMA1->ISR`，但 CubeMX 生成的 `stm32f1xx_it.c` 走 `HAL_DMA_IRQHandler(&hdma_tim3_ch4_up)`。两者冲突——须统一为 **HAL 回调方式**，与生成代码一致。

### 10.1 回调映射

| 事件 | HAL 回调 | 对应动作 |
|------|---------|---------|
| Camera DMA 半满(HT) | `HAL_DMA_XferHalfCpltCallback` | 发 Buffer A (320B) |
| Camera DMA 全满(TC) | `HAL_DMA_XferCpltCallback` | 发 Buffer B (320B) + 判帧末 |
| SPI DMA 发送完成 | `HAL_DMA_XferCpltCallback`（hdma_spi2_tx） | 仅清发送忙标志，不做帧逻辑 |
| VSYNC 下降沿 | `HAL_GPIO_EXTI_Callback` | 帧起始/丢帧 |

> 多个 DMA 共享同名回调，须在回调内用 `hdma->Instance == DMA1_Channel3` 区分 Camera DMA 与 SPI DMA。

### 10.2 统一后的代码框架（替换第 4 章）

```c
void HAL_DMA_XferHalfCpltCallback(DMA_HandleTypeDef *hdma) {
    if (hdma->Instance == DMA1_Channel3 && state == FRAME_CAPTURING) {
        HAL_SPI_Transmit_DMA(&hspi2, CameraBuffer, 320);
        byte_sent += 320;
    }
}

void HAL_DMA_XferCpltCallback(DMA_HandleTypeDef *hdma) {
    if (hdma->Instance == DMA1_Channel3 && state == FRAME_CAPTURING) {
        HAL_SPI_Transmit_DMA(&hspi2, CameraBuffer + 320, 320);
        byte_sent += 320;
        if (byte_sent >= byte_total) state = FRAME_DONE;
    } else if (hdma->Instance == DMA1_Channel5) {
        spi_dma_busy = 0;   // SPI 发送完成，清忙标志
    }
}

void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin != OV7670_VSYNC_Pin) return;
    if (state != IDLE) return;          // 忙时丢帧（见第 12 章）
    state = FRAME_START;
    // ... 见第 11 章 FRAME_START 流程
}
```

## 11. LCD 命令与双 DMA 衔接

每帧前 CPU 阻塞发 `CASET/RASET/RAMWR` 命令设写窗口，再启动 Camera DMA→SPI DMA 流水线连续推像素。命令与像素不混杂（命令期间 DMA 未启动）。

### 11.1 帧时序

```
VSYNC ↓ (state==IDLE)
 ┌─ FRAME_START ─────────────────────────────────┐
 │ 1. /RRST 负脉冲 → 读指针=0                      │
 │ 2. /OE 拉低                                     │
 │ 3. CPU 阻塞发 LCD 命令:                          │
 │      lcd_write_cmd(0x2A); lcd_write_data(caset,4);│
 │      lcd_write_cmd(0x2B); lcd_write_data(raset,4);│
 │      lcd_write_cmd(0x2C);                        │
 │ 4. DC=1, CS=0 保持（准备接收像素流）              │
 │ 5. byte_sent=0; byte_total=40960;               │
 │ 6. 启动 TIM3_CH4 PWM + Camera DMA(640,circular) │
 │ 7. state=FRAME_CAPTURING                        │
 └────────────────────────────────────────────────┘
       │ HT/TC 回调 → SPI DMA 发 320B × 128 次
       ▼
 ┌─ FRAME_DONE (byte_sent>=40960) ────────────────┐
 │ 1. 等最后 SPI DMA 发完 (spi_dma_busy==0)        │
 │ 2. 停 TIM3_CH4 PWM + Camera DMA                 │
 │ 3. /OE 拉高, CS 拉高                            │
 │ 4. state=IDLE                                   │
 └────────────────────────────────────────────────┘
```

### 11.2 时序保证

- CPU 发 LCD 命令期间 Camera DMA 未启动，FIFO 写侧持续存（摄像头不停），无数据丢失。
- SPI DMA 发 320B 耗时 142μs < Camera DMA 填 320B 耗时 222μs，SPI 总能在下个半区就绪前发完，不阻塞 Camera DMA。
- `byte_total=40960` / 320 = 128 次半区 DMA，整数，无尾包。

## 12. 状态机补强

相对第 2.5 节，补强以下几点：

1. **忙时丢帧**：VSYNC 到来时若 `state != IDLE`，直接丢弃本帧（不重启采集），下一帧自然覆盖 FIFO。保证当前帧完整发送，避免半帧撕裂。
2. **帧末同步**：`byte_sent >= byte_total` 置 `FRAME_DONE`，但须等最后半区 SPI DMA 发完（`spi_dma_busy==0`）才停 PWM、拉高 /OE/CS、回 `IDLE`。
3. **`byte_total` 更新**：`40960`（160×128×2），对应 128 次 320B 半区传输。
4. **CameraBuffer 放置**：640 字节放 `.bss`，F103C8T6 20KB RAM 充裕；建议 4 字节对齐。
5. **首次启动**：上电后 `state=IDLE`，首个 VSYNC 即开始第一帧；LCD 初始化（第 7 章）与 OV7670 SCCB 配置（第 6 章）须在使能 VSYNC 中断前完成。

### 12.1 完整状态机

```
IDLE
  │ VSYNC ↓ 且 state==IDLE
  ▼
FRAME_START
  │ /RRST 复位 → /OE 低 → 发 LCD 命令(CASET/RASET/RAMWR)
  │ → DC=1/CS=0 → byte_sent=0,byte_total=40960
  │ → 启动 PWM + Camera DMA(640)
  ▼
FRAME_CAPTURING  ←── HT: SPI DMA 发 A(+320) ; TC: SPI DMA 发 B(+320)
  │ byte_sent >= 40960 ?
  ▼
FRAME_DONE
  │ 等 spi_dma_busy==0 → 停 PWM + Camera DMA
  │ → /OE 高, CS 高
  ▼
IDLE  (期间 VSYNC 来则丢帧)

VSYNC ↓ 且 state≠IDLE  →  丢弃，不动作
```
