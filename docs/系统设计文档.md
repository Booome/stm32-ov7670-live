# 系统设计文档

> 项目：STM32 OV7670 Live  |  MCU：STM32F103C8T6（72MHz，LQFP48）

---

## Part A: 总体设计

### A1. 系统架构概览

```
            ┌──────────────┐       ┌──────────────┐
            │  OV7670      │       │  ST7735 LCD  │
            │  (camera)    │       │  160×128 TFT │
            └──────┬───────┘       └──────▲───────┘
                   │ 8-bit D[7:0]         │ SPI2 18MHz
                   │ PCLK                 │ 4-line serial
            ┌──────▼───────┐       ┌──────┴───────┐
            │  AL422B FIFO  │       │  CPU (FC103) │
            │  384KB 帧缓存 │───────│  RCK 1.44MHz │
            │  HREF 选通写  │ PA0~7 │  DMA1_Ch3    │
            └──────────────┘       │  Circular P→M│
                                   │  DMA1_Ch5    │
                                   │  Normal M→P  │
                                   │  HAL 回调驱动 │
                                   └──────────────┘

数据流: OV7670 → FIFO(写9.8MB/s) → GPIOA->IDR(DMA读1.44MB/s)
       → PipelineBuffer[640] → SPI2-DR → LCD(18MHz)
       ↑ 帧内零拷贝，HT/TC 半区乒乓无等待
```

三个关键设计决策：
1. **双 DMA 乒乓缓冲**：Camera DMA（循环填 PipelineBuffer）与 SPI DMA（分段取 PipelineBuffer 送 LCD），帧内全程零拷贝
2. **FIFO 异步桥接**：OV7670 写速率 9.8MB/s ≫ 读速率 1.44MB/s，写远快于读，永不追尾
3. **RCK 精确调速**：TIM3_CH4 PWM 产生 1.44MHz 读时钟，帧读时间 28.4ms，对 30fps 余量 +17%

### A2. 硬件连接

**OV7670 FIFO 模块**：板上集成 OV7670 + AL422B FIFO，通过 22-pin P1 连接器接入。模块内部 OV7670→FIFO 连接（D[7:0]→DI[7:0]、PCLK→WCK）已由 PCB 完成，MCU 仅操作 P1 引出信号。**所有控制信号均经 P1 引出，无需直接操作芯片引脚。**

| 信号 | 引脚 | 功能 | 方向 |
|------|------|------|------|
| OV7670_D0~D7 | PA0~PA7 | 8-bit 数据总线（FIFO DO0~DO7） | 输入 |
| FIFO_RCK | PB1 → TIM3_CH4 | 1.44MHz PWM 读时钟（P1.18） | 输出 |
| FIFO_OE | PA8 | 输出使能，低有效（P1.20） | 输出 |
| FIFO_WR | PB5 | 写使能，低有效 → NAND 门 HREF 选通（P1.19） | 输出 |
| FIFO_WRST | PB7 | 写指针复位，低有效（P1.21） | 输出 |
| FIFO_RRST | PB6 | 读指针复位，低有效（P1.22） | 输出 |
| OV7670_VSYNC | PA11 → EXTI11 | 帧同步信号，下降沿中断（P1.5） | 输入 |
| OV7670_HREF | PA12 | 行有效（P1.6） | 输入 |
| OV7670_SCL | PB10 | SCCB 时钟（P1.3） | 输出 OD |
| OV7670_SDA | PB11 | SCCB 数据（P1.4） | 双向 |
| OV7670_PWDN | PB3 | 掉电控制（P1.16） | 输出 |
| OV7670_RESET | PB4 | 复位（P1.15） | 输出 |
| OV7670_STROBE | PB0 | 闪光灯/频闪（P1.17） | 输出 |
| **LCD_SCL** | PB13 (SPI2_SCK) | SPI 时钟 18MHz（U3.3） | 输出 |
| **LCD_SDA** | PB15 (SPI2_MOSI) | SPI 主出从入（U3.4） | 输出 |
| **LCD_CS** | PB12 | 片选，低有效（U3.7） | 输出 |
| **LCD_DC** | PB8 | 数据/命令选择（U3.6） | 输出 |
| **LCD_RES** | PB14 | 硬件复位，低有效（U3.5） | 输出 |
| **LCD_BLK** | PB9 | 背光使能（U3.8） | 输出 |

SPI2 配置：主机、8bit、CPOL=0/CPHA=0、MSB first、预分频 /2（36MHz APB1 ÷ 2 = 18MHz）。

### A3. 时钟与频率设计

目标：160×128 RGB565 @ 30fps。

**最低 RCK 频率推导**（FIFO 读口 8-bit 并行，每 RCK 周期读 1 byte）：

| 步骤 | 计算 | 结果 |
|------|------|------|
| 每帧像素数 | 160 × 128 | 20 480 px |
| 每帧字节数 | 20 480 × 2（RGB565 = 16 bit/px） | 40 960 B |
| 每秒字节数 | 40 960 × 30 fps | 1 228 800 B/s |
| 折算 RCK 频率 | 1 228 800 ÷ 1 B/拍 | ≈ 1.23 MHz |

> 上值为理论下限，未含行/帧消隐开销，实际须留余量。

**双 DMA 约束**：SPI DMA 发送半 buffer 的时间必须小于 Camera DMA 填满半 buffer 的时间，即 `T_SPI_320B < T_CAM_320B`：

```
320 × 8 / f_SCK  <  320 / f_RCK  →  f_RCK  <  f_SCK  (SPI @ 18 MHz = 2.25 MB/s)
```

TIM3 时钟：APB1 = 72MHz / 2 = 36MHz，定时器时钟翻倍 = 72MHz。`f_RCK = 72MHz / (ARR+1)`。

| RCK 频率 | ARR | Camera 320B | SPI 320B | 半区余量 | 帧时间余量 |
|----------|-----|-------------|----------|---------|-----------|
| 2.0 MHz | 35 | 160 μs | 142 μs | 18 μs | +63% |
| 1.8 MHz | 39 | 178 μs | 142 μs | 36 μs | +46% |
| **1.44 MHz** | **49** | **222 μs** | **142 μs** | **80 μs** | **+17%** |

**SPI 频率校核**（ST7735 4-line serial 写周期 min=66ns → f_max≈15.15MHz）：

| SPI 频率 | 320B 耗时 | 余量(vs 222μs) | 备注 |
|----------|----------|---------------|------|
| 18 MHz（当前） | 142 μs | 80 μs | 周期 55.5ns < 66ns，略超 ST7735 datasheet（频率超出约 +18%），实测模块多可接受 |
| 15 MHz（理论） | 171 μs | 51 μs | 满足约束，但 APB1=36MHz 下预分频只能 /2→18MHz 或 /4→9MHz |
| 9 MHz | 284 μs | **不满足** | SPI 来不及在下一半区就绪前发完 |

> **结论**：唯一可行的 SPI 预分频为 /2=18MHz。若 ST7735 实测不可靠，需降 RCK 或增大 PipelineBuffer。

**最终选择：RCK = 1.44 MHz**（ARR=49, Pulse=24, PWM mode 2, ≈50% 占空比）

- Camera：帧读时间 28.4ms，对 33.3ms 帧周期余量 4.9ms（+17%）
- SPI：DMA 320B 耗时 222μs，SPI 发送 142μs，余量 80μs（+56%），大幅安全
- 写入速度：XCLK=12MHz → v_write≈9.8MB/s ≫ v_read=1.44MB/s（6.8×），写不致追读

## Part B: 核心数据流

### B1. 双 DMA 流水线

两个 DMA 独立运行、各司其职，帧内 Camera DMA 全程不停，全程零拷贝：

```
TIM3_CH4   Camera DMA (Ch3)     SPI DMA (Ch5)
   │            │                    │
   ▼            ▼                    ▼
[RCK]       GPIOA->IDR ────► PipelineBuffer[640] ──► SPI2-DR ──► LCD
  PWM        Circular              Normal (每次 320B)
            HT/TC 中断触发
```

- **Camera DMA**：DMA1_Ch3, Circular, Peripheral→Memory，RCK 每拍自动搬运 1 字节到 PipelineBuffer[640]
- **SPI DMA**：DMA1_Ch5, Normal, Memory→Peripheral，从 PipelineBuffer 中对应半区发送到 SPI2

**PipelineBuffer 布局（640 bytes = 2 × 320B）：**

```
┌──────────────────────┬──────────────────────┐
│    Buffer A (320B)   │    Buffer B (320B)   │
│   PipelineBuffer[0]    │  PipelineBuffer[320]   │
│         ...          │         ...          │
│   PipelineBuffer[319]  │  PipelineBuffer[639]   │
└──────────────────────┴──────────────────────┘
         ▲HT 中断               ▲TC 中断
    "半满，A 可消费"       "全满，B 可消费"
```

**中断处理：**

```
HT (Half-Transfer) 中断：
  ├── SPI DMA 源地址 = &PipelineBuffer[0]
  ├── 长度 = 320 字节
  └── 启动 SPI DMA 发送

TC (Transfer-Complete) 中断：
  ├── SPI DMA 源地址 = &PipelineBuffer[320]
  ├── 长度 = 320 字节
  └── 启动 SPI DMA 发送
```

**时序：**

```
RCK: ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
     ┃  1.44 MHz，帧内持续输出               ┃
     ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

CAM  ┌──A(320B)──┬──B(320B)──┬──A(320B)──┬──B(320B)──┬...
DMA  │   Camera  │   Camera  │  Camera   │  Camera   │
     └───────────┴───────────┴───────────┴───────────┘
          ▲HT          ▲TC         ▲HT         ▲TC

SPI             [SPI发A]    [SPI发B]    [SPI发A]    [SPI发B]
DMA             320B         320B        320B         320B

                    ▲ SPI DMA 必须在下一个 TC 前完成
```

### B2. FIFO 读写控制

AL422B 是 3M-bit（393,216×8）FIFO 帧存储，作为 OV7670 与 STM32 之间的异步缓冲。

**HREF 自动选通（SN74LVC1G00 NAND 门）**：模块内部 NAND 门实现 `/WE = NAND(HREF, FIFO_WR)`：

| FIFO_WR(PB5) | HREF | /WE | FIFO 行为 |
|-------------|------|-----|----------|
| **HIGH** | HIGH（有效行） | **LOW** | **写入有效像素** |
| **HIGH** | LOW（行消隐） | HIGH | 禁止写，跳过消隐无效数据 |
| LOW | 任意 | HIGH | 禁止写，全局关闭 |

STM32 只需将 FIFO_WR 拉高即可自动按 HREF 选通写入，完全无需 MCU 参与 HREF 时序。**溢出分析**：VGA 模式下每帧 ≤ 307,200 字节 < FIFO 容量 393,216 字节，无溢出风险。

**VSYNC 帧时序**（OV7670 默认极性）：

```
              ←────────── 帧总周期 510 t_LINE ──────────→

              ←3→←── 17 t_LINE ─→←──── 480 t_LINE ────→←10→
              同步   垂直后肩        有效行 (HREF=高)    前肩
VSYNC: ━━━━━┓                              ┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
            ┃ 高=同步脉冲(~3 LINE)         ┃ 低=帧有效传输(~507 LINE)   ┃
            ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛                           ┗
            ↑↑ 上升沿                       ↑↓ 下降沿: 同步结束,帧数据区开始 ↑ 下一帧
            │                               │
            │                               └─ HREF 在 17 LINE 后首次拉高
            └─ 同步脉冲（仅 3 LINE, @12MHz ≈196μs）

HREF: ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓  ┏┓  ┏┓  ┏━━━━━━━━━━━━━━━━━━━━━━
                 ← 后肩: HREF=低 →     ┃  ┃┃  ┃┃  ┃
                 NAND 门输出高,无写入   ┗━━┛┗━━┛┗━━┛
                                      row0 row1 row2 ...
```

垂直后肩时长：`t_后肩 = 17 × 784 × t_PCLK = 13328 / PCLK`。@12MHz：1.11ms。

写入速率：`v_write = PCLK × 640 / 784 ≈ PCLK × 0.816`。@12MHz ≈ 9.80 MB/s。读速率 1.44 MB/s，写/读 ≈ 6.8×。

**写操作流程：**

```
VSYNC ↓ 中断（同步结束→帧开始）
  → FIFO_WRST(PB7) 负脉冲 → 写指针=0
  → FIFO_WR(PB5) 拉高 → NAND 门工作
  → HREF 在 1.11ms 后首次拉高
  → 有效行(HREF=高) → /WE=低 → PCLK 上升沿写入 FIFO
  → 行消隐(HREF=低) → /WE=高 → 禁止写入
VSYNC ↓（下一帧）→ FIFO_WR 拉低，准备读操作
```

**读操作流程（延迟启动策略）：**

```
VSYNC ↓ 中断
  ├─ 写侧: /WRST → FIFO_WR=1
  ├─ 延时 ≈1.3ms（> 后肩 1.11ms, 确保 FIFO 中已有数据）
  ├→ /RRST 负脉冲 → 读指针=0
  ├→ /OE 拉低（持续使能输出）
  ├→ 启动 TIM3_CH4 PWM(RCK 1.44MHz) + Camera DMA(640,circular)
  ├→ HT/TC 中断触发 SPI DMA 发送
  ├─ ~28.4ms：byte_sent 累计到 40960 → 停止
  └→ /OE 拉高，等待下一个 VSYNC↓
```

延时精度：1.3ms vs 1.11ms 后肩，余量约 190μs。须用 DWT CYCCNT（Cortex-M3 周期计数器，@72MHz 精度 14ns）实现，不可用 SysTick。

**AC 时序参数（datasheet 7.4，3.3V 模式）：**

| 参数 | 含义 | Min | Max |
|------|------|-----|-----|
| T_RC | RCK 周期 | 20 ns | 1000 ns |
| T_RPH / T_RPL | RCK 高/低脉宽 | 7 ns | — |
| T_AC | 访问时间（RCK↑→数据有效） | — | 15 ns |
| T_OH | 输出保持 | 4 ns | — |
| T_RRS / T_RRH | /RRST 建立/保持 | 5 / 2 ns | — |
| T_OES / T_OEH | /OE 建立/保持 | 5 / 2 ns | — |

RCK=1.44MHz → 周期 694ns ≫ T_RC(20ns)；T_AC(15ns) 内数据就绪，余量充足。

**关键约束（datasheet 8.3）：**

1. WCK/RCK 须保持 ≥1MHz 自由运行作 DRAM 刷新。帧内 RCK 1.44MHz 持续，满足。
2. 读地址须滞后写地址 128~393,247 周期。VSYNC 同步下读指针复位为 0，写指针从 0 增长，满足。
3. ⚠ `/RE` 不能在 `/RRST` 上升沿为低（datasheet 8.3.2）。本项目模块 /RE 恒接地，且 `/RRST` **每帧读前均拉低再释放**——每次 `/RRST` 上升沿均违反此约束。

> **约束 3 说明**：此为模块 PCB 硬件设计决定，/RE 引脚固化接地，**软件和硬件层面均无法修复**。
>
> 违规机理（datasheet 8.3.2）：`/RRST` 上升沿触发读地址指针复位初始化，此时若 `/RE` 也为低（读使能），内部 DRAM 状态机可能产生"一边复位一边读"的竞态。datasheet 配图明确标记此为禁止情况。
>
> 实际稳定性：该模块有一定出货量且工作稳定，推测原因——(a) `/RRST` 上升沿时 RCK 未启动（PWM 处于关闭态），无有效读时钟边沿触发实际读取；(b) `/OE` 在 `/RRST` 复位后且启动 PWM 前才拉低，复位瞬时输出高阻；(c) 即使读指针瞬态异常，后续正式读出的第一帧数据仍因指针小幅漂移（而非完全不可控）而可接受。此约束以 **"已知偏差 — 无修复方案 — 实测稳定"** 记录，不再列为待解决项。

### B3. 帧采集状态机

```
                           VSYNC ↓ 且 state≠IDLE
                        ┌──────────────────────────────
                        │  （DISABLED 态：直接返回，未初始化）
                        │  （忙时丢帧，下一帧自然覆盖 FIFO）
                        │
PIPELINE_STATE_IDLE         ▼
  │   VSYNC ↓ 且 state==IDLE
  ▼
PIPELINE_STATE_FRAME_START
  │ 1. /WRST 负脉冲 → FIFO 写指针=0
  │ 2. FIFO_WR 拉高 → NAND 门自动 HREF 选通
  │ 3. 延时 ≈1.3ms（> 后肩 1.11ms @12MHz, DWT 非阻塞等待）
  │ 4. /RRST 负脉冲 → FIFO 读指针=0
  │ 5. /OE 拉低
  │ 6. LCD_SetAddrWindow（CASET/RASET/RAMWR，CS=0, DC=1 保持）
  │ 7. s_bytes_sent=0; 帧大小=40960 (160×128×2)
  │ 8. 启动 TIM3_CH4 PWM + Camera DMA(640, circular)
  │ 9. state=PIPELINE_STATE_FRAME_CAPTURING
  ▼
PIPELINE_STATE_FRAME_CAPTURING  ←── 循环
  │ HT 回调 → SPI DMA 发 Buffer A (320B), s_bytes_sent+=320
  │ TC 回调 → SPI DMA 发 Buffer B (320B), s_bytes_sent+=320
  │          → s_bytes_sent >= 40960 ? state=PIPELINE_STATE_FRAME_DONE
  ▼
PIPELINE_STATE_FRAME_DONE
  │ 1. 等最后 SPI DMA 发完（s_spi_dma_busy==false）
  │ 2. 停 TIM3_CH4 PWM + Camera DMA
  │ 3. /OE 高, CS 高, FIFO_WR 低
  ▼
PIPELINE_STATE_IDLE


帧间: VSYNC ↓ → /WRST 复位写指针 → FIFO_WR=1 → 延时 1.3ms
              → /RRST 复位读指针 → 读操作 → 下一帧

初始态（上电复位）:
PIPELINE_STATE_DISABLED = 0（默认）→ Pipeline_Init() → PIPELINE_STATE_IDLE
```

关键保障：
- **忙时丢帧**：VSYNC 到来时若 state≠IDLE（含 DISABLED 态），丢弃本帧（不重启采集），保证当前帧完整
- **帧末同步**：须等最后半区 SPI DMA 发完（`s_spi_dma_busy==false`）才停 PWM、回 IDLE
- **整数分区**：PIPELINE_FRAME_SIZE=40960 / 320 = 128 次半区 DMA，无尾包
- **PipelineBuffer 放置**：640 字节放 `.bss`，F103C8T6 20KB RAM 充裕；建议 4 字节对齐
- **首次启动**：上电 `s_state=0=PIPELINE_STATE_DISABLED`（BSS 零初始化）；`Pipeline_Init()` 切换至 IDLE；首个 VSYNC 即开始第一帧

### B4. RGB565 零转换

OV7670 配 RGB565（COM15 bit[5:4]=01），ST7735 配 16bit/pixel RGB565（COLMOD=0x05）。两者像素格式一致，PipelineBuffer 中每 2 字节即 1 个 RGB565 像素，SPI DMA 直接搬到 LCD 显存，**无需任何颜色/字节序转换**。

字节序匹配：OV7670 默认高字节在前（MSB first），ST7735 4-line serial 16bit 模式亦高字节在前，SPI 配 MSB first。三者一致，直送即可。

> 若实测颜色错乱（如红蓝互换）：检查 OV7670 COM3 bit6（MSB/LSB swap）、ST7735 MADCTL D3（RGB/BGR）。调整其一即可，仍不需软件转换。

>

>
> 推荐使用 `HAL_DMA_Start_IT(&hdma_tim3_ch4_up, (uint32_t)OV7670_DATA_ADDR, ...)` 直接指定源地址。

### C2. OV7670 SCCB 配置

**目标分辨率与缩放策略**：160×128 通过 OV7670 两层缩放架构实现，无裁剪、无黑边。

| 步骤 | 水平 | 垂直 |
|------|------|------|
| VGA 原始 | 640 | 480 |
| Down Sampling by 2 (DCW) | 320 | 240 |
| Digital Zoom Out | → 160（0.5x） | → 128（0.533x） |

缩放公式（IG Section 6.4）：

```
SCALING_XSC = 0x20 × (DownSample_Width / Target_Width) = 0x20 × 320/160 = 0x40
SCALING_YSC = 0x20 × (DownSample_Height / Target_Height) = 0x20 × 240/128 = 0x3C
```

其中 0x20 → 1.0x（不缩放），0x40 → 0.5x。验证：320×0.5=160, 240×(32/60)=128 ✓。

**SCCB 协议时序**（兼容 I²C，STM32 GPIO 模拟）：

| 参数 | 含义 | 要求 |
|------|------|------|
| f_SIO_C | 时钟频率 | ≤ 400 kHz |
| t_LOW | 时钟低电平 | ≥ 1.3 μs |
| t_HIGH | 时钟高电平 | ≥ 600 ns |

写寄存器：`START → 0x42(ID+W)+ACK → reg_addr+ACK → data+ACK → STOP`

读寄存器：`START → 0x42+ACK → reg_addr+ACK → RESTART → 0x43(ID+R)+ACK → data(NACK) → STOP`

> 72MHz 下，使用 DWT_DelayCycles 实现 cycle-accurate 延时：t_LOW = 100 cycles (1.39μs)，t_HIGH = 50 cycles (0.69μs)，加 GPIO 开销约 ~190 cycles/bit ≈ 379kHz（≤ 400kHz 规格）。

**寄存器配置表（160×128 RGB565）：**

完整配置共 158 条寄存器，基于 Linux 内核 ov7670.c 默认值 + bayernfan CameraInit + LiveOV7670 交叉验证。以下按功能分组列出关键寄存器，完整表见 `ov7670.c` 中 `s_reg_config[]`。

**缩放与输出格式（本项目特有）：**

| 地址 | 寄存器 | 写入值 | 作用 |
|------|--------|--------|------|
| 0x12 | COM7 | 0x14 | bit4=1 QVGA（DCW by2->320×240）；bit[2:0]=100 -> RGB565 |
| 0x0C | COM3 | 0x0C | bit3=1 Digital Zoom Enable；bit2=1 Down Sampling Enable |
| 0x3E | COM14 | 0x18 | bit4=1 DCW/scaling PCLK 使能；bit3=1 手动缩放使能 |
| 0x40 | COM15 | 0xD0 | bit[7:6]=11 输出满量程 [00-FF]；bit[5:4]=01 RGB565 |
| 0x72 | SCALING_DCWCTR | 0x11 | bit[5:4]=01 垂直 by2；bit[1:0]=01 水平 by2 |
| **0x70** | **SCALING_XSC** | **0x40** | **水平 0.5x**（320->160） |
| **0x71** | **SCALING_YSC** | **0x3C** | **垂直 0.533x**（240->128） |
| 0x73 | SCALING_PCLK_DIV | 0xF1 | PCLK /2（匹配 DCW by2；D[7:4]=0xF 为 reserved 位必须值） |

**自动曝光 / 白平衡 / 增益：**

|------|--------|--------|------|
| 0x13 | COM8 | 0xE0->0xE5->0xE7 | 三段写入：先禁用 AGC/AEC/AWB 设参数，再依次开启 AGC+AEC，最后开启 AWB |

>
> **验证要点**：写完寄存器后，用彩条测试模式验证每帧输出 160×128×2 = 40960 字节。通过后再关闭彩条进入正常采集。

### C3. ST7735 LCD 驱动

**上电初始化序列**（datasheet 9.12/9.13）：

1. LCD_BL 拉高开背光
2. LCD_RESET 拉低 ≥10μs 后拉高（硬件复位）
3. 延时 120ms（寄存器恢复）
4. `SLPOUT (0x11)` 退出睡眠，延时 120ms（等 booster 稳定）
5. `MADCTL (0x36)` + 参数：内存访问方向
6. `COLMOD (0x3A)` + 0x05：16bit/pixel RGB565
7. `FRMCTR1 (0xB1)` + 帧率参数（可选）
8. `INVON (0x21)` 或 `INVOFF (0x20)`（按模块极性，1.8寸常需 INVON）
9. `NORON (0x13)`：正常显示模式
10. `DISPON (0x29)`：开显示

**MADCTL(0x36) 位定义：**

| bit | 名称 | 含义 |
|-----|------|------|
| D7 | MY | 行地址方向 |
| D6 | MX | 列地址方向 |
| D5 | MV | 行/列交换 |
| D4 | ML | LCD 刷新方向 |
| D3 | RGB | 0=RGB, 1=BGR |

MADCTL 取值按屏幕安装方向调整。160×128 横屏时设 MV=1 交换行列。若模块像素为 BGR 排列则 D3=1。

**帧写入协议**（每帧前 CPU 阻塞发命令，与像素 DMA 流不混杂）：

```
1. CASET(0x2A) + 4字节: XS_H XS_L XE_H XE_L    // 列 0~159
2. RASET(0x2B) + 4字节: YS_H YS_L YE_H YE_L    // 行 0~127
3. RAMWR(0x2C)                                  // 开始接收像素数据流
```

> 发命令时 DC=0；发参数与像素时 DC=1。RAMWR 之后 DC 保持 1，ST7735 连续接收 40960 字节像素。地址寄存器在写窗口内自动递增。

**与 DMA 的衔接**——每帧时序：

```
VSYNC ↓ (state==IDLE)
  ┌─ FRAME_START ───────────────────────────────────┐
  │ ... FIFO 读写侧启动（见 B3）                       │
  │ 延时 1.3ms 后 →                                     │
  │ CPU 阻塞发 LCD 命令:                              │
  │   lcd_write_cmd(0x2A); lcd_write_data(caset,4);   │
  │   lcd_write_cmd(0x2B); lcd_write_data(raset,4);   │
  │   lcd_write_cmd(0x2C);                            │
  │ DC=1, CS=0 保持（准备接收像素流）                  │
  │ 启动 TIM3_CH4 PWM + Camera DMA(640,circular)     │
  │ state=FRAME_CAPTURING                            │
  └──────────────────────────────────────────────────┘
       │ HT/TC 回调 → SPI DMA 发 320B × 128 次
       ▼
  FRAME_DONE → 等 SPI DMA 完成 → 停 PWM → /OE 高, CS 高

时序保证:
  - CPU 发 LCD 命令期间 Camera DMA 未启动，FIFO 写持续存，无数据丢失
  - SPI 发 320B(142μs) < Camera 填 320B(222μs)，SPI 不阻塞 Camera
  - 40960/320 = 128 次，整数
```

## Part D: 软件实现

### D1. 初始化与启动流程

**外设初始化顺序**（对应 main.c）：

```
HAL_Init()
SystemClock_Config()    // HSE 8MHz → PLL×9 = 72MHz
MX_GPIO_Init()          // 所有引脚初始电平 + VSYNC EXTI 配置 + 立即禁用 EXTI IRQ
MX_DMA_Init()           // DMA1 时钟 + NVIC 中断使能
MX_TIM3_Init()          // TIM3 PWM 配置（RCK 初始停止）
MX_SPI2_Init()          // SPI2 18MHz + 链接 hdma_spi2_tx
MX_USART1_UART_Init()   // 调试串口 115200

// 用户代码区 (USER CODE BEGIN 2):
Pipeline_EnableTimDma();   // 启用 TIM3 CC4 DMA 请求

// BSP 初始化（顺序严格）:
1. DWT_Init();          // DWT CYCCNT 启用，须在所有延时调用前
2. OV7670_Init();       // PWDN/RESET 硬件上电 + COM7 软复位 + 内部 SCCB_Init + 158 寄存器 SCCB 配置
                         //   失败则调用 Error_Handler 终止
3. LCD_Init();          // ST7735 上电序列（复位 + SLPOUT + MADCTL + DISPON）
4. Pipeline_Init();     // 状态机初始化 -> state = PIPELINE_STATE_IDLE
5. Pipeline_ClearVsyncPending();  // 清除挂起的 VSYNC 中断
6. Pipeline_EnableVsyncIrq();     // 使能 OV7670_VSYNC_EXTI_IRQn（PA11 -> EXTI11，最后一步）
```

> 关键：DWT 初始化必须最先，SCCB 和 LCD 延时均依赖 DWT CYCCNT。`SCCB_Init()` 已收归 `OV7670_Init()` 内部调用，`main.c` 不再直接接触 SCCB 传输层。`MX_GPIO_Init` 在配置 EXTI 后立即通过 USER CODE 调用 `HAL_NVIC_DisableIRQ` 禁能 VSYNC 中断，确保 OV7670 和 LCD 初始化期间不会被 VSYNC 误触发。`Pipeline_Init` 将状态机从 `PIPELINE_STATE_DISABLED(0)` 切换至 `PIPELINE_STATE_IDLE(1)`——即使 VSYNC 中断在 DISABLED 态误触发，`Pipeline_OnVsync()` 的 `(s_state != IDLE)` 守卫也会直接返回。

**首次启动时序：**

```
上电 -> HAL/MX 外设初始化 -> DWT_Init -> OV7670_Init(含 SCCB) -> LCD_Init
-> Pipeline_Init -> 使能 VSYNC 中断
-> 首个 VSYNC ↓ -> PIPELINE_STATE_FRAME_START -> /WRST -> FIFO_WR=1
-> DWT 非阻塞等待 1.3ms (DWT_DelayStart/DelayExpired)
   └─ 此期间 FIFO 已有约 5.6KB 数据（9.8MB/s × 1.3ms × 0.816/2）
-> /RRST -> /OE L -> LCD_SetAddrWindow -> 启动 PWM+DMA -> PIPELINE_STATE_FRAME_CAPTURING
-> 128 次半区传输 -> PIPELINE_STATE_FRAME_DONE -> IDLE -> 等待下一个 VSYNC
```

### D2. HAL 回调与中断

CubeMX 生成的 `stm32f1xx_it.c` 使用 `HAL_DMA_IRQHandler` 处理中断，回调统一为 HAL 风格。

**DMA 句柄引用（与代码一致）：**
- Camera DMA: `hdma_tim3_ch4_up` (DMA1_Channel3)
- SPI DMA: `hdma_spi2_tx` (DMA1_Channel5)

**回调映射：**

| 事件 | HAL 回调 | 对应动作 |
|------|---------|---------|
| Camera DMA 半满(HT) | `hdma.XferHalfCpltCallback`（函数指针） | 发 Buffer A (320B) + s_bytes_sent+=320 |
| Camera DMA 全满(TC) | `hdma.XferCpltCallback`（函数指针） | 发 Buffer B (320B) + s_bytes_sent+=320 + 判帧末 |
| SPI DMA 发送完成 | `HAL_SPI_TxCpltCallback`（weak 覆盖） | 清 s_spi_dma_busy 标志 |
| VSYNC 下降沿 | `HAL_GPIO_EXTI_Callback`（weak 覆盖） | 帧起始/丢帧 |

> Camera DMA 使用函数指针回调（hdma.XferHalfCpltCallback / XferCpltCallback），
> SPI DMA 使用 HAL_SPI_TxCpltCallback weak 函数覆盖。两者独立，无需区分通道。

**代码框架：**

```c
/* Camera DMA 回调（函数指针，read_start() 中设置） */
static void pipeline_dma_half_cplt_cb(DMA_HandleTypeDef *hdma) {
    Pipeline_OnDmaHalfCplt();  /* Buffer A [0..319] -> SPI DMA */
}

static void pipeline_dma_cplt_cb(DMA_HandleTypeDef *hdma) {
    Pipeline_OnDmaCplt();      /* Buffer B [320..639] -> SPI DMA + 判帧末 */
}

/* SPI DMA 发送完成（HAL weak 函数覆盖） */
void HAL_SPI_TxCpltCallback(SPI_HandleTypeDef *hspi) {
    Pipeline_OnSpiDmaCplt();   /* 清 s_spi_dma_busy 标志 */
}

/* VSYNC 下降沿（HAL weak 函数覆盖） */
void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin) {
    if (GPIO_Pin != OV7670_VSYNC_Pin) return;
    Pipeline_OnVsync();        /* 帧起始或丢帧 */
}
```

**VSYNC 延时实现（DWT 非阻塞）：**

```c
/* pipeline.c 模块内部 */
static DWT_DelayHandle s_vsync_delay;

void Pipeline_OnVsync(void) {
    if (s_state != PIPELINE_STATE_IDLE) return;  /* DISABLED 或忙时丢帧 */
    s_state = PIPELINE_STATE_FRAME_START;
    OV7670_FIFO_WRST_Low(); OV7670_FIFO_WRST_High();  /* /WRST 负脉冲 */
    OV7670_FIFO_WR_High();                           /* FIFO_WR=1 */
    DWT_DelayStart(&s_vsync_delay, 1300u);            /* 1.3ms 非阻塞 */
}

/* 主循环轮询 */
void Pipeline_Poll(void) {
    if (s_state == PIPELINE_STATE_FRAME_START) {
        if (DWT_DelayExpired(&s_vsync_delay)) {
            read_start();  /* /RRST -> /OE L -> LCD_SetAddrWindow -> PWM+DMA */
        }
    }
}
```

`read_start()` 内容：/RRST -> /OE L -> LCD_SetAddrWindow -> PWM+DMA 启动 -> state=PIPELINE_STATE_FRAME_CAPTURING。

> 延时精度：1.3ms vs 1.11ms 后肩余量 190μs。DWT @72MHz 精度 14ns ≪ 余量，远满足。SysTick 默认 1ms tick 精度不足（±1ms 抖动可能完全错过后肩），不可用。
